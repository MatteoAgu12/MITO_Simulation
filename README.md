# Come lavorare

* Per ogni nuova feature o modifica creiamo un nuovo branch in local in cui facciamo tutto quello che vogliamo.
* Stando sul branch possiamo fare tutte le modifiche e i commit che ci pare
* Quando siamo soddisfatti dell'implementazione apriamo una merge request
* Per ogni merge fare anche update del file README

# Programmazione
Voglio un aprogramamzione fortemente object oriented (quasi ogni cosa deve essere una classe).
* Ogni classe e metodo deve essere ben documentato
* Opzionale ma carino: sezione di testing per le funzioni più complesse
